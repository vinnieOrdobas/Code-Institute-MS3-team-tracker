Team Tracker

What is it
Team Tracker is a custom-made cloud-based solution intended for teams to track their training records in an intuitive interface - it is a mobile-first app, and its layout is simple, dynamic and visually appealing. There are no requirements for its usage: no need for heavy software or any external installation packages. It is meant to replace obsolete processes that are created as spreadsheets that are neither user-friendly or efficient - it's a time saver for companies/teams of every size. Team Tracker is a free-to-use, dynamic app powered by mongoDB.
How it works
Being cloud-based, Team Tracker runs in any device that can access the internet - its wired up with mongoDB, which runs a backend server to keep all the information secure and ready to use.
Why is it relevant
Team Tracker saves time by presenting records in an interactive way - its dynamic structure is user-friendly, which means users do not need to dig for information, saving time across the board.
Implementation
The first step is to identify the need to keep track of training needs for your team - think about all of those 4000-row spreadsheets you currently have in your organisation. To install Team Tracker, please refer to the documentation and the GitHub repository.
Using it
After the solution is implemented, the only step needed before start using the system is to register users - that can be done through the Registration page. To create a System Administrator, please refer to the documentation.
Access tiers

Team Tracker is organised in access-tiers, where different types of users can perform different actions - this framework allows for a seamless experience for all types of users, and caters for a diverse range of internal clients. For more information, check the table below.

The System Administrator is the user who'll provide the structure for Team Tracker - this user can create/edit/delete teams, trainings, cycles and edit user's access. There can be only one per system.

Cycle System
Team Tracker uses a Cycle-based framework to break-up trainings in up to three different stages: the so-called cycles. Within this framework, each cycle has one particular function:
  • - Training Deck:
    • This is the initial training program, where the instructor presents the theoretical material on an electronic format, generally through slides and describes the training.
  • - Shadowing:
    • This is a hands-on cycle, where the instructor works through training in a practical fashion, leading the way.
  • - Reverse Shadowing:
    • In this final cycle, the positions are inverted and the student works through the training with the aid of the instructor.

About

Team Tracker stemmed from the idea of optimization. Being part of one of those inefficient, time-consuming and clunky processes drove me to create a system which would prevent time loss. Using these'user-hostile' processes hijacked over 35% of my time in the office - I was in desperate need of optimizing; so I went out to brainstorm with my colleagues. The shape of the system, its layout, functionalities and colour scheme are the children of this brainstorm sessions.
