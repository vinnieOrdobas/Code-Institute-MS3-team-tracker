What is it

Team Tracker is a custom-made cloud-based solution intended for teams to track their training records in an intuitive interface - it is a mobile-first app, and its layout is simple, dynamic and visually appealing. There are no requirements for its usage: no need for heavy software or any external installation packages. It is meant to replace obsolete processes that are created as spreadsheets that are neither user-friendly or efficient - it's a time saver for companies/teams of every size. Team Tracker is a free-to-use, dynamic app powered by mongoDB.

How it works

Being cloud-based, Team Tracker runs in any device that can access the internet - its wired up with mongoDB, which runs a backend server to keep all the information secure and ready to use.

Why is it relevant

Team Tracker saves time by presenting records in an interactive way - it's dynamic structure is user-friendly, which means that members of teams do not need to dig for information, and saves times across the board.

Implementation

The first step is to identify the need to keep track of training needs for your team - think about all of those 4000-row spreadsheets you currently have in your organisation. To install Team Tracker, please refer to the documentation and the GitHub repository.

Using it

After the solution is implemented, the only step before start using the system is to register users - that can be done through the Registration page. To create a System Administrator, please refer to the documentation.

Access tiers

Team Tracker is organised in access-tiers, where different types of users can perform different actions - this framework allows for a seamless experience for all types of users, and caters for a diverse range of internal clients. For more information, check the table below.

System Administrator Team Leader Instructor Student

System Administrator

The System Administrator is the user who'll provide the structure for Team Tracker - this user can create/edit/delete teams, trainings, cycles and edit user's access. There can be only one per system.

Agree

Team Leader

The Team Leader is the manager of the process - this user has access to add/edit/delete trainings and cycles within his team, enroll students in different training sets and check student graduations.

Agree

Instructor

The Instructor has the same tier of access as the Team Leader, with the difference that he works cross-teams - he's the one who's get his name printed on Cycle cards and on Student's Certificates.

Agree

Student

The Student is star of this system: the app was created for him. Although he doesn't have CRUD functionalities with his access team, the system records Student's certification in it's own training folder.

Agree
